## 三、关键设计决策

### 3.1 Invoice 金额

- `subtotal`：行项目合计
- `taxRate` / `taxAmount`：可选税
- `totalAmount`：应收总额（与 PRD Amount 一致）
- 金额统一 `Decimal(18,2)`，避免浮点误差

### 3.2 软删除

- `clients.deletedAt`、`invoices.deletedAt`：列表默认过滤，保留财务审计

### 3.3 状态与索引

- `invoices` 复合索引：`(userId, status)`、`(status, dueDate)` 供 Dashboard / Cron 扫描逾期
- `invoice_number` 全局唯一

### 3.4 并发编号

`invoice_sequences` + 事务递增，服务见 `src/lib/services/invoice-number.service.ts`。

### 3.5 Business 预留

- `invoices.organizationId` 可空，个人用户为 null
- `api_keys` 仅组织级，阶段 10 后接 API 网关

## 四、Migration 命令

```bash
cp .env.example .env.local
# 编辑 DATABASE_URL

pnpm install
pnpm db:migrate    # 开发：创建 migration
# 或
pnpm db:migrate:deploy  # 生产：仅应用已有 migration

pnpm db:seed       # 可选：演示数据
pnpm db:studio     # 可视化查看
```

## 六、订阅限额查询（实现参考）

```sql
-- Free 计划：当月 Invoice 数量
SELECT COUNT(*) FROM invoices
WHERE user_id = $1
  AND created_at >= date_trunc('month', NOW())
  AND deleted_at IS NULL;
```

Pro/Business 以 `subscriptions.plan` 为准。
